Marine Organism Levy Flight Simulation - Manual
1. Project Overview
This project simulates the Levy flight migration patterns of marine organisms in a global ocean environment. The simulation includes:

Global migration capabilities with longitude/latitude boundary wrapping

Temperature-weighted direction selection (optional)

Dual Levy exponent experiments for comparative analysis

Multiple agent behaviors with supply, lifespan, and boundary limitations

Interactive visualization with real-time plotting and data export

Full reproducibility through explicit random seed control

5. Input File Specifications
5.1 Map File (merged_result.xlsx)
Format: Excel (.xlsx) file

Grid Size: 180 rows × 360 columns (global scale: 1°×1° resolution)

Cell Values:

"G" : Coastal/Shallow water (Green zone - full supply)

"B" : Open ocean (Blue zone - limited or no supply)

"Y" : Land (Yellow zone - impassable)

"<angle>-G" : Shallow water with current direction (e.g., "90-G")

"<angle>-B" : Open ocean with current direction (e.g., "45-B")

5.2 Temperature File (Optional for Weighted Mode)
Format: Excel (.xlsx) file

Grid Size: Must match map dimensions (180×360)

Values: Temperature in degrees Celsius

Usage: Used for temperature-weighted direction selection (50% weight)

6.2 Simulation Settings (Configurable via GUI)
Number of Agents: 1-1000 agents (default: 500)

Maximum Steps: Simulation steps (default: 100,000)

Agent Speed: Self-propelled movement speed (m/s)

Starting Location: Latitude/Longitude coordinates

Pure Survival Time: Days agents can survive without supply

Direction Mode: "weighted" (50% random + 50% temperature) or "random"

Current Influence: "with_current" or "no_current"

Global Simulation: Enable/disable boundary wrapping

Lifespan Limit: Enable/disable maximum agent lifespan

B-Zone Supply: Enable/disable partial supply in B zones

6.3 Random Seed Configuration (Critical for Reproducibility)
The simulation uses two independent random seeds to ensure complete reproducibility:

Direction Seed:

Controls the random direction selection for agents

When left blank in the GUI, the program will generate a random seed and display it in the console output

Can be set to any integer value for reproducible direction patterns

Console output format: 方向种子留空，自动生成基础种子：123456（记录可复现） or 已设置基础方向种子：123456（方向选择可复现）

Levy Seed:

Controls the Levy step length (time t) generation for agents

When left blank in the GUI, the program will generate a random seed and display it in the console output

Can be set to any integer value for reproducible step length sequences

Console output format: Levy种子留空，自动生成基础种子：654321（记录可复现） or 已设置基础Levy步长种子：654321（步长t可复现）

Key Features of Seed Implementation:

Each agent uses a derived seed from these base seeds (base seed + agent index) to ensure independence while maintaining reproducibility

Seeds are saved in the configuration file (simulation_config.json) for future sessions

Seeds are exported in the Excel summary sheet for complete documentation

Both seeds are displayed in the interactive results window for immediate reference

8. Key Features
8.1 Global Wrapping Migration
Enabled: Agents can cross ±180° longitude boundaries

Disabled: Agents die when reaching map edges

Implementation: Uses coordinate normalization and path segmentation

Visualization: Correct path plotting across date line

8.2 Temperature-Weighted Direction Selection
Mode: Weighted (50% random + 50% temperature-weighted)

Limitation: Requires complete temperature grid (180×360)

Scoring: Converts temperature to movement preference scores

Note: This feature is primarily theoretical due to lack of complete Mesozoic temperature data

8.3 Ocean Currents
Representation: Directional vectors embedded in map cells

Influence Mode: Can be enabled/disabled

Speed: Fixed at 0.1 m/s (configurable)

Effect: Vector addition with agent's self-propelled movement

8.4 Agent Lifecycle
Birth: Start at specified coordinates

Movement: Levy flight with directional constraints

Supply Management:

G zones: Full supply reset

B zones: Partial or no supply (configurable)

Y zones: Impassable (land)

Death Conditions:

Supply depletion (starvation)

Lifespan expiration (if enabled)

Boundary crossing (if global mode disabled)

10. Experimental Considerations
10.1 Temperature Data Limitation
Important Note: The temperature-weighted direction feature is included for completeness but has limited practical application in Mesozoic research due to:

Data Scarcity: Complete global sea surface temperature datasets for the Triassic period are rare

Data Resolution: Available data often lacks the required 1°×1° resolution

Temporal Gaps: Discontinuous temporal coverage in geological records

Model Uncertainty: High uncertainty in paleoclimate reconstructions

Recommendation: Use "random" direction mode for most experimental scenarios unless high-quality temperature data is available.

10.2 Dual Levy Experiments
Purpose: Compare migration patterns under different Levy exponents

Interpretation:

Lower exponents (<2): Longer "super-diffusive" movements

Higher exponents (>2): More localized exploration

Exponent=2: Brownian motion (standard diffusion)

Scientific Value: Useful for studying optimal foraging strategies

10.3 Reproducibility and Random Seeds
Seed Management: The simulation uses two separate seeds for direction and Levy step generation, allowing independent control of these two stochastic processes.

Automatic Seed Generation: If seeds are not provided, the program will generate random seeds and display them in the console output. These seeds are also saved in the exported data file.

Replication: To exactly replicate a simulation, note the seeds from the console output or exported data and enter them in the GUI for subsequent runs.

Seed Documentation: Always record the seeds used in experiments for scientific reproducibility.

10.4 Experimental Workflow Recommendations
Pilot Studies: Run simulations with auto-generated seeds to establish baseline parameters

Formal Experiments: Use fixed seeds for reproducible results

Parameter Sweeps: Vary one parameter at a time while keeping seeds constant

Documentation: Always include seed values in experimental records and publications

11. Troubleshooting
11.1 Common Issues
Issue	Solution
"Map size mismatch" error	Ensure map and temperature files have identical dimensions (180×360)
Tkinter not found (Linux)	Install python3-tk package (see Installation section)
Memory errors with many agents	Reduce agent count or maximum steps
Slow performance	Disable real-time rendering for large simulations
Missing Excel file error	Place merged_result.xlsx in project directory
Plotting errors	Ensure matplotlib backend is set to TkAgg
Non-reproducible results	Check that both Direction Seed and Levy Seed are set to specific values
11.2 Performance Optimization
Large Simulations (>1000 agents or >500k steps):

Disable real-time visualization

Increase refresh interval

Reduce path sampling frequency

Memory Management:

Enable garbage collection (gc.collect())

Close figures after use

Use numpy arrays instead of Python lists
